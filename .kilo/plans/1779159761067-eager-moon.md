# Plan: Strip & Simplify Barony for WASM Target

## Goal

Prepare the Barony codebase for eventual WASM deployment (no Emscripten) by:
1. Removing all non-Windows platform code
2. Removing proprietary SDK integrations (Steam, EOS, PlayFab)
3. Removing unnecessary optional dependencies (imgui, TheoraPlayer, cURL, Opus, Tremor)
4. Replacing RapidJSON with a small C JSON library
5. Converting C++ to pure C (keeping audio working throughout)

**Critical constraint: Audio must remain working at every stage.** The game uses FMOD (C++ API). We convert it to FMOD's C API to keep it working during C++→C conversion. Audio is the LAST thing replaced (with Web Audio API for WASM).

**Scope**: ~487K lines, 158 `.cpp` files, 55 `.hpp` files, 3 `.h` files (accurate count after all stripping phases)

---

## Audio Preservation Strategy

The audio system stays working through the entire migration:

| State | Phase | Audio State | How |
|---|---|---|---|---|
| Current | — | Working | FMOD C++ API (fmod.hpp) |
| C++→C conversion | 9.3 | Working | FMOD C++ API → FMOD C API (fmod.h) — same .dll |
| Pure C | 9.4–9.5 | Working | FMOD C API via `zig cc` linking fmod.dll |
| C→Zig replacement | 11 | Working | C audio called from Zig via `@cImport("fmod.h")` |
| WASM target | 12 | Replaced | FMOD stubbed; Web Audio API backend in Zig |

**FMOD's C API** (`fmod.h`) is a first-class interface, not a wrapper — it provides the same functionality as the C++ API (`fmod.hpp`). The conversion is mechanical:
```cpp
// C++ API (current)
FMOD::System* system;
system->createSound(name, mode, 0, &sound);
system->playSound(sound, 0, false, &channel);

// C API (converted)
FMOD_SYSTEM* system;
FMOD_System_CreateSound(system, name, mode, 0, &sound);
FMOD_System_PlaySound(system, sound, 0, 0, &channel);
```

### 6.3a Critical issue discovered during conversion: cJSON for-loop increment

During the rapidjson→cJSON conversion of `GameUI.cpp` and `interface.cpp`, a systematic bug was introduced: **all `for`-loop increments were left as `++itr` (pointer arithmetic) instead of being changed to `itr = itr->next` (linked-list traversal).**

**Root cause:** RapidJSON iterators use `operator++()` to advance through an internal array (proper iterator semantics). cJSON stores child items as a **singly-linked list**, so advancing requires `current = current->next`. Using `++itr` on a `cJSON*` just does `itr += sizeof(cJSON)` — jumping to whatever heap data happens to be at that offset.

**Scope:** 37 occurrences across 2 files were fixed:
- `GameUI.cpp`: 24 fixes — `itr`, `val_itr`, `inner_itr`, `ally_itr`, `entry_itr`, `pot_itr`, `itr2`, `options_itr`, `effPos_itr`, `arr`
- `interface/interface.cpp`: 13 fixes — `itr`, `itr2`, `itr3`, `highlightItr`

**Warning for future conversions:** Every `for` loop that iterates `->child` in cJSON MUST use `var = var->next`, never `++var`.

### 6.3b Bug discovered during final mod_tools conversion: premature cJSON_Delete

During the final rapidjson→cJSON conversion of `mod_tools_tooltips.cpp`, a use-after-free bug was introduced: **`cJSON_Delete(d)` was placed right after the items-reading loop but before the spells-reading loop** (which still needed `d`).

**Root cause:** RapidJSON's `Document` is stack-allocated with RAII destructor — no manual cleanup needed. cJSON requires explicit `cJSON_Delete(d)` after the **last** use of the parsed tree. The converter placed the delete after the first section without noticing the function continued using `d`.

**Fix:** Move `cJSON_Delete(d)` to after the last use of `d` (after the spells loop) in `readItemsFromFile`.

**Lesson:** When converting from rapidjson to cJSON, `cJSON_Delete` must be placed at the **very end** of the function, not after the first parse result is consumed.

## Phase 9: C++ → C Language Conversion ⏳ PENDING

### Strategy: Bottom-up dependency order

The dependency graph was analyzed by tracing `#include "..."` transitively through the `.hpp` chain. Key finding: **`main.hpp` is a god header included by every file**. Before starting C conversion, break it apart to avoid cascading breakage.

### 9.0 Prerequisite: Fix god headers

**Goal**: Break `main.hpp` and other wide headers into focused sub-headers so converting a utility file to C doesn't force converting the entire system.

```
main.hpp → break into:
  ├── defs.h          — Basic types, constants, forward declarations (C-compatible)
  ├── game_types.h    — Game-specific type declarations
  └── main.h          — Convenience umbrella #include-ing the above (keep for transition)
```

Other wide headers to consider:
- `draw.hpp` (depended on by 8 hpp files) — split draw declarations from UI
- `consolecommand.hpp` (depended on by 10 hpp files) — mostly forward decls, keep focused

**C11 threads**: Replace SDL_mutex/SDL_thread with C11 `<threads.h>` (`thrd_t`, `mtx_t`, `cnd_t`). Available in `zig cc` with `-std=c11`. Reduces SDL2 dependency for WASM.

Files to convert: `src/engine/audio/sound.hpp`, `src/defines.cpp`, `src/init_audio.cpp`, `src/sound.cpp`, `src/sound_game.cpp`, `src/music.cpp`.

**Do this before converting audio files to C** so the `.cpp` files stay compilable during the transition.

### 9.6 Per-file conversion checklist

For each `.cpp` file being converted to `.c`:

1. [ ] Rename `.hpp` includes → `.h`
2. [ ] Replace `#include <fmod.hpp>` → `#include <fmod.h>` (audio files only)
3. [ ] `std::string` → `String` from ccontainers.h
4. [ ] `std::vector` → `Array(T)` from ccontainers.h
5. [ ] `std::map` / `std::unordered_map` → `Map` / `HashMap` from ccontainers.h
6. [ ] Other std:: containers → C equivalents
7. [ ] `auto` → explicit type
8. [ ] Range-based `for` → index/pointer loop
9. [ ] Lambda → named function + `void* context`
10. [ ] `static_cast`/`const_cast`/`reinterpret_cast` → C casts
11. [ ] `new`/`delete` → `malloc`/`free`
12. [ ] `nullptr` → `NULL`
13. [ ] `constexpr` → `#define` or `static const`
14. [ ] Default params → wrapper or `#define`
15. [ ] `enum class` → `enum`
16. [ ] `class` → `struct`
17. [ ] Function overloading → unique names (`foo_int`, `foo_float`, etc.)
18. [ ] Constructor/destructor → `_init`/`_deinit` functions
19. [ ] `template` → macro or `void*`
20. [ ] `thread_local` → `_Thread_local`
21. [ ] SDL_mutex/SDL_thread → C11 `mtx_t`/`thrd_t`
22. [ ] Rename file → `file.c`, header → `file.h`
23. [ ] Update `build.zig` — change .cpp to .c, ensure compiles
